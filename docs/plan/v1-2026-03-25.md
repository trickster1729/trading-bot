# Trading Bot — Project Plan v1
# Saved: 2026-03-25 (first draft, imported from initial design session)

## Overview

An algorithmic trading system targeting consistent returns across Stocks, Crypto, and Options.
Built in phases: starting as a local CLI with shadow/backtesting, graduating to a cloud-hosted
live trading engine. All layers are designed to be pluggable so strategies, brokers, and models
can be swapped without rewriting core logic.

---

## Goals

- Target: 5–10%+ monthly returns with controlled drawdown
- Starting capital (once live): $5,000–$10,000
- Non-goal: maximise raw return at the expense of stability — risk management is first-class
- Long-term: cloud-hosted, fully automated, multi-asset, multi-strategy

---

## Asset Classes

- Stocks / Equities
- Crypto
- Options / Derivatives

---

## Build Phases

### Phase 1 — CLI + Shadow Trading (current)
- Local Python CLI
- Historical data replay (Yahoo Finance for stocks, CCXT for crypto)
- Paper PnL tracking — no real money
- One baseline strategy per asset class (e.g. momentum)
- Backtest harness with performance report (Sharpe, drawdown, win rate)
- Goal: validate the pipeline end-to-end and prove at least one profitable signal

### Phase 2 — Paper Live Trading
- Connect to live market data feeds (real prices, not historical)
- Alpaca paper account (stocks/options) + Binance testnet (crypto)
- Same code as Phase 1, just swap data source and execution target
- Run continuously for 4–8 weeks, log every trade decision
- Gate: must show positive expectancy over 100+ trades before advancing

### Phase 3 — Small Live Trading
- Flip Alpaca + Binance from paper to live accounts
- Start with $5,000–$10,000 real capital, strict position size limits
- Hard kill-switch: halt all trading if drawdown exceeds 10% of account
- Weekly performance review before increasing position sizes

### Phase 4 — Scale + Cloud
- Containerise with Docker, deploy to AWS ECS (Fargate)
- Add IBKR for options volume and margin
- Add Coinbase Advanced as Binance fallback
- Multi-strategy portfolio (uncorrelated signals running in parallel)
- Full observability: PnL dashboard, trade log, alerting (PagerDuty or SNS)

## Architecture

```
Data Ingestion Layer
  ├── Yahoo Finance (stocks, free), CCXT (crypto), Alpaca WebSocket (live)

Signal Engine (Strategy Layer)
  ├── Strategy base class — pluggable interface: generate_signal(market_data) → Signal
  └── Implementations: Momentum, MeanReversion, StatArb, LLMSentiment, RLAgent

Risk Management Layer
  ├── Position sizing (Kelly criterion or fixed fractional)
  ├── Max drawdown guard (hard stop)
  ├── Stop-loss per trade
  └── Correlation limits across open positions

Execution Layer (Broker Abstraction)
  ├── Broker base class — pluggable interface: submit_order(signal) → OrderResult
  └── Implementations: AlpacaBroker, BinanceBroker, IBKRBroker, PaperBroker

Simulation / Live Toggle
  ├── MODE=shadow  → historical replay, paper PnL
  ├── MODE=paper   → live prices, paper orders
  └── MODE=live    → live prices, real orders

Logging Layer
  ├── Structured JSON logs via Python `structlog`
  ├── Every trade decision logged: signal, confidence, price, quantity, broker response
  ├── Every risk check logged: what was evaluated, what was blocked and why
  ├── Log levels: DEBUG (local dev), INFO (paper), WARNING/ERROR (live)
  └── Log sink: local file (Phase 1), CloudWatch Logs (Phase 3+)

Monitoring & Alerting Layer
  ├── Metrics: PnL, win rate, Sharpe, drawdown, signal frequency, order latency
  ├── Health checks: data feed alive, broker connection alive, strategy loop running
  ├── Alerts: drawdown threshold breach, broker API error, missed heartbeat
  ├── Alert channels: console (Phase 1), email/SMS via AWS SNS (Phase 3+)
  └── Dashboard: CLI report command (Phase 1), Grafana or CloudWatch dashboard (Phase 4)
```

---

## Tech Stack

| Layer | Choice | Reason |
|---|---|---|
| Language | Python 3.11+ | Best quant ecosystem |
| CLI | Typer | Clean, typed, easy to extend |
| Data (stocks) | yfinance + Alpaca | Free historical + live |
| Data (crypto) | CCXT | Unified API across 100+ exchanges |
| Backtesting | VectorBT or Backtrader | Fast vectorised replay |
| Broker (stocks) | Alpaca → IBKR | Simple REST API, free paper trading |
| Broker (crypto) | Binance → Coinbase Adv. | Best liquidity, testnet available |
| Database | SQLite → PostgreSQL | Simple to start, scalable later |
| Logging | structlog | Structured JSON logs, consistent format across all layers |
| Log sink (local) | File + Rich console | Human-readable in dev, JSON file for analysis |
| Log sink (cloud) | AWS CloudWatch Logs | Centralised, searchable, retention policies |
| Metrics | Prometheus + Grafana (Phase 4) | PnL, latency, win rate dashboards |
| Alerting | AWS SNS → email/SMS | Drawdown breaches, broker errors, missed heartbeats |
| Health checks | Custom heartbeat loop | Verifies data feed, broker connection, strategy loop |
| Cloud | AWS ECS (Fargate) | Familiar, cost-effective at this scale |
| Infra-as-code | Terraform or AWS CDK | Reproducible deployments |
| Secrets | AWS Secrets Manager | Never commit credentials |

---

## Risk Rules (non-negotiable from day 1)

- Max 2% of account per trade
- Max 10% drawdown — halt all trading, alert
- No leverage in Phase 1–2
- No overnight positions in Phase 1 (equities)
- All trades logged with full context (signal, confidence, entry, exit, PnL)
- Kill switch: `bot stop --force` halts all open orders immediately

---

## AI / Model Layer

- **Baseline (Phase 1–2):** classical quant signals — momentum, RSI, Bollinger Bands, MACD
- **Sentiment layer (Phase 3):** news API + Claude API — per-ticker sentiment score
- **ML signals (Phase 3):** feature engineering on OHLCV + indicators — Random Forest / XGBoost
- **RL agent (Phase 4):** PPO or SAC agent trained on historical data
